What I Believe

  • Architecting UI should be the first step in building websites
  • This should be done by a single empowered systems architect, or a small cohesive group
  • Consistent component design allows you to build websites that have conceptual integrityâ€”an important feature of elegance in design
  • The web has a finite number of design constraints, and so UI design is convergent.
  • Because design converges starting from UI prototypes and iterating through progressive refinement is superior to starting from a blank canvas
  • This applies to all design (buildings, system design, film)

For The Startup

  • Most of creating websites is building layouts (production)
  • For smaller startups, compentized design through UI planning is critical because
    • High volatilityof staffing (people come and go)
    • Your team needs to build super fast.
    • Empower junior developers, first time
  • For larger companies (and future unicorns) singular UI is also critical.
    • Lots of disparate products need one unified brand
    • Currently 2.9 billion internet users visit 968,000 websites.
    • Thats a lot of iteration on design principles.
